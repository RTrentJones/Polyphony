# Provenance — Frankenstein (renamed eval corpus)

## Source work
- **Title:** *Frankenstein; or, The Modern Prometheus*
- **Author:** Mary Wollstonecraft Shelley
- **First published:** 1818
- **Copyright status:** **Public domain in the United States** (published before 1929).
- **Text source:** Project Gutenberg edition (eBook #84). Project Gutenberg's own
  license states that once the Project Gutenberg header, license, and all
  references to the Project Gutenberg trademark are removed, *"you are left with a
  text unrestricted by U.S. intellectual property law."* `evals/tools/rename.py`
  strips that wrapper (`strip_gutenberg`) before any text is used. Equivalent
  CC0 editions are available from Standard Ebooks (CC0 1.0) and may be used
  interchangeably.

## What lives here
- **`excerpts.txt`** — a **renamed derivative** of six sections (Letters 1–4,
  Chapter 5, Chapter 13), not the whole novel. Generated by
  `python -m evals.tools.rename --book frankenstein`.
- **`aliases.json`** — the rename map + the sections extracted.
- **`ground_truth.json`** — labels derived from the renamed text
  (`python -m evals.tools.build_gt --book frankenstein`): cast, per-character gold
  voice lines (train/test), canonical plot beats, continuity-injection specs.

## Why this book
Frankenstein is the **held-out validation corpus**. Like Dracula it has multiple
**disjoint first-person narrators** — Walton (the framing letters), Victor (his
creation chapter), and the creature (its own recounted tale) — so the
voice-separability premise holds and the same graders apply. But it is a
different cast, register, and structure, so a prompt/architecture change that only
overfits Dracula's ground truth will **not** score here. An improvement counts
only if it holds on both books.

## Why the names are changed
Every character and place is renamed (Victor Frankenstein → Emeric Vandel, Robert
Walton → Rowan Merrow, Geneva → Verenna, …). This is for **evaluation integrity
only**: feeding a model the *original* text measures how well it recalls a famous
novel from training, not how well Polyphony's pipeline extracts characters,
grounds voice, and plans plot from the text in front of it. Renaming defeats that
memorization while preserving the (public-domain) prose and the ground truth.
**It is not an attempt to obscure the source** — this file credits it plainly.

## A note on the gold-line cap
The creature's chapter is rendered as a nested multi-paragraph quotation (each
paragraph opens with `“`); `build_gt.py`'s `quoted` flag keeps that framed
narration as voice lines instead of discarding it as dialogue. Walton's letters
yield far more lines than the other two narrators, so a per-narrator `max_lines`
cap balances the pool — an imbalanced pool biases retrieval toward the majority
voice and understates separability. Balanced, the three voices separate at
precision@3 ≈ 0.85 (well above the 0.33 three-class chance baseline).

## Legal summary
The underlying work is US public domain; the Project Gutenberg wrapper is stripped
(or a CC0 Standard Ebooks source is used); only a renamed derivative of a few
sections is committed — never the raw source, never the whole novel. A renamed
derivative of a public-domain work is itself unrestricted. Only this
public-domain-derived text is ever sent to an LLM during evals.
